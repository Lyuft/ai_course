# Локальная инфраструктура

Этот каталог содержит Docker Compose-конфигурацию для локального запуска PostgreSQL и Liquibase.

Инфраструктура нужна только для разработки и проверки учебного проекта на компьютере студента. Production-развертывание в этом репозитории не описывается.

## Что такое Docker

Docker позволяет запускать приложения в изолированных контейнерах.

Базовые понятия:

* **Образ** - шаблон приложения со всем необходимым окружением. Например, `postgres:16-alpine` уже содержит PostgreSQL 16.
* **Контейнер** - запущенный экземпляр образа. Один и тот же образ можно запускать много раз, получая разные контейнеры.
* **Dockerfile** - инструкция для сборки собственного образа. В этой задаче он не нужен, потому что PostgreSQL и Liquibase берутся из готовых публичных образов.
* **Volume** - постоянное хранилище данных контейнера. Без volume данные контейнера считаются временными.

В этом учебном проекте Docker нужен, чтобы студентам не приходилось вручную устанавливать PostgreSQL, создавать базу, схему и выполнять SQL-скрипты.

## Установка Docker

На macOS обычно достаточно установить Docker Desktop с официального сайта, и дополнительных проблем чаще всего не возникает.

На Windows установка может потребовать дополнительной настройки. Рекомендуемый вариант - Docker Desktop с WSL 2. Подробная инструкция:

https://selectel.ru/blog/tutorials/docker-desktop-windows/

Действуйте по инструкции. Если Docker Desktop с WSL 2 никак не удается настроить, можно попробовать чистую установку Docker Desktop с Hyper-V. Это запасной вариант, но на некоторых Windows-системах он оказывается проще.

## Что такое Docker Compose

Docker Compose - это инструмент для запуска нескольких связанных контейнеров одной командой.

В этом каталоге файл `docker-compose.yml` описывает два сервиса:

* `postgres` - контейнер с PostgreSQL 16;
* `liquibase` - одноразовый контейнер, который применяет миграции к базе данных.

Команда:

```bash
docker compose up -d
```

делает сразу несколько вещей:

1. читает файл `docker-compose.yml`;
2. скачивает недостающие Docker-образы;
3. создает и запускает контейнер PostgreSQL;
4. дожидается готовности PostgreSQL через `healthcheck`;
5. запускает контейнер Liquibase;
6. применяет миграции и тестовые данные;
7. оставляет PostgreSQL работать в фоне.

Флаг `-d` означает detached mode: контейнеры запускаются в фоне, а терминал освобождается.

Проверить состояние контейнеров:

```bash
docker compose ps
```

Посмотреть логи Liquibase:

```bash
docker compose logs liquibase
```

Остановить инфраструктуру:

```bash
docker compose down
```

Все команды нужно выполнять из каталога `infra`, потому что именно здесь лежит `docker-compose.yml`.

## Что такое Liquibase

Liquibase - это инструмент для управления структурой базы данных через миграции.

Миграция описывает, какие изменения нужно применить к базе: создать таблицу, добавить ограничение, вставить начальные данные и так далее. Вместо ручного выполнения SQL-скриптов студент запускает Compose, а Liquibase сам приводит базу к нужному состоянию.

Почему это удобно:

* структура базы хранится в репозитории рядом с проектом;
* запуск одинаковый у всех студентов;
* не нужно вручную создавать таблицы и роли;
* меньше ошибок из-за пропущенных SQL-команд;
* можно повторить чистую инициализацию базы в любой момент.

В этом проекте Liquibase запускается отдельным контейнером. Java-приложение не должно подключать Liquibase и не должно выполнять миграции при старте.

## Чистый локальный запуск

Мы специально не подключаем постоянный volume для PostgreSQL.

Это сделано для учебной задачи: каждый чистый запуск должен быть предсказуемым. Если удалить контейнеры и поднять инфраструктуру заново, база создается с нуля, миграции применяются повторно, тестовые данные загружаются заново.

В реальных проектах данные PostgreSQL обычно хранят в volume, например:

```yaml
volumes:
  - postgres_data:/var/lib/postgresql/data
```

Но для этого задания постоянное хранилище не требуется. Если подключить volume, старые данные могут сохраниться между запусками и помешать проверке чистого сценария.
